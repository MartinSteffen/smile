Vorläufige Projektstruktur:

Ich habe ein Eclipse-Projekt "smile" angelegt mit getrennten Verzeichnissen
für Sources und Classfiles.

----
Vorläufiger Inhalt des Wurzelverzeichnisses:

README.txt
    Diese Datei soll vorläufig für Notizen dienen, die alle etwas angehen.
    Macht damit, was ihr wollt.

src
    Das Source-Verzeichnis

bin
    Das Verzeichnis für die Classfiles

javadoc
    Das Verzeichnis für die von javadoc generierte HTML-Dokumentation
    (alle Inhalte werden von svn ignoriert, da die Property svn:ignore
    auf '*' gesetzt wurde. Für handgeschriebene Dokumentation wird man
    also ein anderes Verzeichnis verwenden müssen)

Dokumentation
    handgeschriebene Dokumentation

----
Vorläufige Package-Struktur

(default package)
Main.java
    das Hauptprgramm, das die GUI aufruft

csm
    Die Core-State-Machine
    (gehört allein der Data-Structure-Gruppe)

gui
    Die GUI
    (gehört allein der GUI-Gruppe)

nua
    Der \nu-Automat
    (gehört allein der Data-Structure-Gruppe)

testing
    Sammelstelle für alle Unit-Tests
    (gehört allein der Testing-Gruppe)


----

bescheidenen Vorschlag darstellen.
